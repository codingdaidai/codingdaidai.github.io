面向对象 继承

概述

概念：让类与类之间产生父子关系

​ 被继承的类叫做父类

​ 继承的类叫做子类

格式：（entends）

​ class 父类{

​ //…

​ }

​ class 子类 extends 父类 {

​ //…

​ }

效果：子类拥有了父类的非私有成员（成员变量，成员方法）

继承的使用场景

当多个类存在相同属性和行为时，可以将这些内容提取出来放到一个新类中，让这些类和新类产生父子关系，实现代码复用

继承的优缺点

  • 优点：
    • 功能复用
    • 便于扩展新功能
    • 结构清晰，简化认识
    • 易于维护
  • 缺点：
    • 打破了封装性：父类向子类暴露了实现细节
    • 高耦合性：类与类紧密结合，相互依赖性高

程序设计要求

低耦合，高内聚 （内聚：模块内部结构紧密，独立性强）

继承关系中类成员的使用

子类和父类中定义了相同的成员变量，如何使用

使用变量遵循就近原则：局部位置有就使用，没有就去本类的成员位置找，再没有就去父类的成员位置找

当子父类的变量重名时：

使用this，指定本类的成员变量；

使用super，指定父类的成员变量

查找顺序：局部变量-本类的成员变量-父类-更高的父类…Object

对象初始化顺序：先初始化父类内容，再初始化子类内容

继承中成员方法的使用：就近原则，与变量相同

访问父类方法的方式：super.父类方法名（）；当父类功能不能完全满足需求时使用

继承中子父类构造方法的使用： 当父类中自定义了带参构造，子类在创建对象时，需手动调用。super（参数）；

this和super的区别

  • 本质：对象；父类内存空间的标识
  • 用法：从本类开始找；从父类开始找

方法重写（Override）

  • 定义：子类中出现和父类方法定义相同的方法的现象。方法名，参数列表，返回值类型都相同

  • 注意：父类的私有方法无法重写；

    ​ 子类方法访问权限不能小于父类方法

  • 用途：扩展父类功能；

    ​ 父类功能过时时，重新实现父类功能

Java中的访问权限修饰符：

本类 同包下的类 不同包下的子类 不同包下的无关类
private √
默认 √ √
protected √ √ √
public √ √ √ √

  • private：强调给自己来使用；

  • 默认：强调给同包下的使用；

  • protected：强调给子类使用；

  • public：强调给大家使用

方法重写和方法重载的区别

overload override
方法名 相同 相同
参数列表 不同 相同
返回值类型 无关 相同
修饰符 无关 访问权限不小于重写方法
定义位置 同一类中 子父类中

Java中继承的特点

  • 单继承：只支持类的单继承，但是支持多层继承（只能有一个爸爸，但可以有很多儿子）（另：接口支持多继承）
  • 私有成员不能继承：子类只能继承父类的非私有成员
  • 构造方法不能继承：构造方法用于初始化本类对象，创建子类对象时，需要调用父类构造初始化该对象的父类内容
  • 继承体现了“is a ”的关系： 子 is a 父
