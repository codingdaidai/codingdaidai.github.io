接口

接口概述

概念：用于描述类具有什么功能，但不给出具体实现，类要遵循接口描述的统一规则进行定义，所以，接口是对外提供的一组规则，标准。

定义：使用关键字interface

​ interface 接口名{}

​ 类和接口是实现关系，用implements表示：

​ class 类名 implements 接口名

接口创建对象的特点：接口不能实例化，通过多态的方式实例化子类对象；接口的子类可以是抽象类，也可以是普通类。若是普通类，必须重写接口中所有抽象方法，若是抽象类，不用重写

接口继承关系的特点：

  • 接口与接口之间的关系：继承关系，可以多继承，格式：

    ​ 接口 extends 接口1，接口2，…

  • 继承与实现的区别：继承体现的是“is a ”关系，父类中定义共性内容；实现体现的是“like a ”关系，接口中定义扩展功能

接口成员的特点

  • 接口没有成员变量，只有公有的，静态的常量：

    ​ public static final 常量名 = 常量值；

  • 成员方法：公有的，抽象方法：public abstract 返回值类型 方法名（）；

    ​ 默认方法和静态方法：public default 返回值类型 方法名（）{}

    ​ static 返回值类型 方法名（）{}

    ​ 私有方法：private 返回值类型 方法名（）{}

  • 构造方法：无
