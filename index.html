实用指令

指令运行级别

  • 0：关机
  • 1：单用户（找回丢失密码）
  • 2：多用户无网络服务
  • 3：多用户有网络服务
  • 4：保留
  • 5：图形界面
  • 6：重启

常用的运行级别是3，5，要修改默认的运行级别可修改配置文件

系统的运行级别配置文件：/etc/inittab (注意：centos7不支持），可修改开机运行的默认级别。centos7可以直接用set-default :将默认运行级别设置为指定的target；

centos7不再使用/etc/inittab文件进行默认的启动级别配置，而使用比sysvinit的运行级更为自由的target替代。第3运行级用multi-user.target替代，第5运行级用graphical.target替代。

centos7运行级别

centos7运行级别设置

  • 运行级别对应关系

    init level       systemctl target
0            shutdown.target
1            emergency.target
2            rescure.target
3            multi-user.target
4            无
5            graphical.target
6            reboot.target

  • 运行级别设置

    • 语法：systemctl [command] [unit.target]

    • 命令及参数

      • command部分

        get-default :获取当前的target；

        set-default :将默认运行级别设置为指定的target；

        isolate :切换至指定的运行级别。

      • unit.target部分：为上面1节部分中给出的运行级别

  • 常用运行级别相关命令

    • systemctl
      • systemctl get-default：获取当前的运行级别；
      • systemctl set-default multi-user.target：将默认运行级别设置为mulit-user；
      • systemctl isolate multi-user.target：不重启系统的情况下，将运行级别切换至mulit-user；
      • systemctl isolate graphical.target：不重启系统的情况下，将运行级别切换至图形模式
    • 其他：
      • runlvel：返回结果中，第一个数为之前运行级别，后一个数为当前运行级别；
      • init 5：将当前运行级别切换至5（图形模式）

集合

集合简介

what：用来存储多个元素的容器

集合和数组的区别

  • 元素类型：引用类型（存储基本类型时自动装箱）；基本类型，引用类型
  • 元素个数：不固定，可以任意扩容；固定，不能改变容量
  • 集合的好处：不受容器大小限制，可以随时添加，删除元素，提供了大量操作元素的方法（判断，获取等）

Java的集合体系：

  • 单列集合（Collection）（接口）
    • List（子体系）–ArrayList（子类）
    • Set–HashSet
  • 双列集合（Map：key,value）
    • HashMap（子类）

List集合的特点和应用

特点：可重复，有序（存取顺序相同）

应用：List list = new ArrayList();

List接口中的成员方法：

  • public boolean add(E e)：将数据添加到集合末尾，E是泛型
  • public E get(int index)：根据索引取得对应元素
  • public int size()：获取集合长度

使用集合的步骤：

  1. 创建集合对象
  2. 创建元素对象
  3. 将元素对象添加到集合对象中
  4. 遍历集合

增强for循环和迭代器

why：简化数组和集合的遍历

格式：for(数据类型 变量名 ：数组或集合对象){

​ //循环体

​ }

why迭代器：是遍历Collection集合的通用方式，可以在遍历的同时进行添加，删除等操作

常用方法：

  • next()：返回迭代的下一个元素对象
  • hasNext()：如果仍有元素可以迭代，则返回true

使用步骤：

  1. 根据集合对象获取其对象的迭代器对象
  2. 判断迭代器中是否有元素
  3. 如果有就获取元素

接口

接口概述

概念：用于描述类具有什么功能，但不给出具体实现，类要遵循接口描述的统一规则进行定义，所以，接口是对外提供的一组规则，标准。

定义：使用关键字interface

​ interface 接口名{}

​ 类和接口是实现关系，用implements表示：

​ class 类名 implements 接口名

接口创建对象的特点：接口不能实例化，通过多态的方式实例化子类对象；接口的子类可以是抽象类，也可以是普通类。若是普通类，必须重写接口中所有抽象方法，若是抽象类，不用重写

接口继承关系的特点：

  • 接口与接口之间的关系：继承关系，可以多继承，格式：

    ​ 接口 extends 接口1，接口2，…

  • 继承与实现的区别：继承体现的是“is a ”关系，父类中定义共性内容；实现体现的是“like a ”关系，接口中定义扩展功能

接口成员的特点

  • 接口没有成员变量，只有公有的，静态的常量：

    ​ public static final 常量名 = 常量值；

  • 成员方法：公有的，抽象方法：public abstract 返回值类型 方法名（）；

    ​ 默认方法和静态方法：public default 返回值类型 方法名（）{}

    ​ static 返回值类型 方法名（）{}

    ​ 私有方法：private 返回值类型 方法名（）{}

  • 构造方法：无

面向对象 继承

概述

概念：让类与类之间产生父子关系

​ 被继承的类叫做父类

​ 继承的类叫做子类

格式：（entends）

​ class 父类{

​ //…

​ }

​ class 子类 extends 父类 {

​ //…

​ }

效果：子类拥有了父类的非私有成员（成员变量，成员方法）

继承的使用场景

当多个类存在相同属性和行为时，可以将这些内容提取出来放到一个新类中，让这些类和新类产生父子关系，实现代码复用

继承的优缺点

  • 优点：
    • 功能复用
    • 便于扩展新功能
    • 结构清晰，简化认识
    • 易于维护
  • 缺点：
    • 打破了封装性：父类向子类暴露了实现细节
    • 高耦合性：类与类紧密结合，相互依赖性高

程序设计要求

低耦合，高内聚 （内聚：模块内部结构紧密，独立性强）

继承关系中类成员的使用

子类和父类中定义了相同的成员变量，如何使用

使用变量遵循就近原则：局部位置有就使用，没有就去本类的成员位置找，再没有就去父类的成员位置找

当子父类的变量重名时：

使用this，指定本类的成员变量；

使用super，指定父类的成员变量

查找顺序：局部变量-本类的成员变量-父类-更高的父类…Object

对象初始化顺序：先初始化父类内容，再初始化子类内容

继承中成员方法的使用：就近原则，与变量相同

访问父类方法的方式：super.父类方法名（）；当父类功能不能完全满足需求时使用

继承中子父类构造方法的使用： 当父类中自定义了带参构造，子类在创建对象时，需手动调用。super（参数）；

this和super的区别

  • 本质：对象；父类内存空间的标识
  • 用法：从本类开始找；从父类开始找

方法重写（Override）

  • 定义：子类中出现和父类方法定义相同的方法的现象。方法名，参数列表，返回值类型都相同

  • 注意：父类的私有方法无法重写；

    ​ 子类方法访问权限不能小于父类方法

  • 用途：扩展父类功能；

    ​ 父类功能过时时，重新实现父类功能

Java中的访问权限修饰符：

本类 同包下的类 不同包下的子类 不同包下的无关类
private √
默认 √ √
protected √ √ √
public √ √ √ √

  • private：强调给自己来使用；

  • 默认：强调给同包下的使用；

  • protected：强调给子类使用；

  • public：强调给大家使用

方法重写和方法重载的区别

overload override
方法名 相同 相同
参数列表 不同 相同
返回值类型 无关 相同
修饰符 无关 访问权限不小于重写方法
定义位置 同一类中 子父类中

Java中继承的特点

  • 单继承：只支持类的单继承，但是支持多层继承（只能有一个爸爸，但可以有很多儿子）（另：接口支持多继承）
  • 私有成员不能继承：子类只能继承父类的非私有成员
  • 构造方法不能继承：构造方法用于初始化本类对象，创建子类对象时，需要调用父类构造初始化该对象的父类内容
  • 继承体现了“is a ”的关系： 子 is a 父

  2. 第一行是创建学生类的对象，而内存中没有这个类，于是找到学生类的字节码文件，加载到内存；
  3. 在栈中创建一个学生类的引用Stu，看到new会向堆内存申请开辟空间，初始的name（string类型）为null，age（int类型）为0，study方法指向方法区，此时会把0x0011这个地址赋给栈中的stu引用；
  4. 再执行给对象赋值的操作，name和age改了；
  5. 执行方法：根据0x0011找到堆中的该块空间，然后根据0x0066找到该方法，从而study方法进栈；
  6. study方法第一句也有string name的值，为“小黑”，对方法本身已经有的变量，不再去找成员变量，于是输出“小黑正在努力学习！”

Java中使用变量的规则

  • 成员位置：写在类中，方法外；
  • 局部位置：方法体语句当中，或者方法的参数列表（即在方法内部的）

遵循就近原则，如果局部位置有，直接使用；如果没有，就去本类的成员位置找

成员变量和局部变量的区别：

  • 定义位置：类中，方法外；方法中或形式参数
  • 初始化值：有默认初始化值（null，0）；无默认初始化值，必须先赋值再使用
  • 作用范围：在类中；在方法中
  • 内存中位置：堆内存中；栈内存中
  • 生命周期：随着对象创建而存在，随对象消失而消失；随方法调用而存在
  • 注意事项：如果成员变量和局部变量重名，采用就近原则

构造方法：也叫构造器，用来帮助创建对象的方法，作用是初始化对象

如何创建对象：通过new关键字创建对象，并在内存中开辟空间，然后使用构造方法完成对象的初始化

构造方法格式：修饰符 构造方法名（参数列表）{

​ //方法体

​ }

构造方法要求：方法名与类名相同；无返回值；无返回值类型

注：不提供构造方法，系统会给默认无参构造；构造方法可重载；若提供了构造方法，系统不再提供

标准代码：JavaBean

编写类的标准规范，必须是具体的公共的，且具有无参数的构造方法，提供用于操作private成员变量的set和get方法。

封装的关键：绝不能让类中的方法访问其他类的数据（属性），程序仅通过对象的方法和对象的数据进行交互。
